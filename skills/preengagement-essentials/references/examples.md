# Pre-Engagement Essentials Examples — Real User Triggers

## Example 1: Start a real pentest correctly
**User:** "Start a real pentest engagement"
**Expected:** Confirm authorization, scope, ROE, constraints, approvals, contacts, and data-handling rules before active testing.

## Example 2: Explain scope vs ROE
**User:** "What's the difference between scope and ROE?"
**Expected:** Explain scope as what/where, ROE as how/when/under what constraints.

## Example 3: Check if we're actually allowed to test
**User:** "Can we test this target yet?"
**Expected:** Verify signed permission, boundaries, third-party/cloud approvals, and stop if authorization is incomplete.

## Example 4: Build a pentest ROE
**User:** "Help me write the rules of engagement"
**Expected:** Produce an ROE structure covering timing, allowed/prohibited actions, contacts, stop conditions, data handling, and reporting.

## Example 5: Handle third-party/cloud concerns
**User:** "This is in AWS/Azure / hosted by a provider — what do we need before testing?"
**Expected:** Check provider policies, third-party approvals, and scope boundaries before testing.

## Example 6: Safety and SOC coordination
**User:** "How do we avoid turning the pentest into an incident?"
**Expected:** Set safe windows, communications, emergency contacts, incident procedures, and resume authority.

## Example 7: Scope change discipline
**User:** "We found another environment mid-test — can we include it?"
**Expected:** Route through explicit scope change or document as limitation, not quiet scope creep.
