# Report Phase Essentials Examples — Real User Triggers

## Example 1: Make the report stronger
**User:** "Write the pentest report properly"
**Expected:** Produce a multi-audience report with executive narrative, reproducible findings, remediation roadmap, and QA discipline.

## Example 2: Cleanup must be in the report
**User:** "Make sure the report includes cleanup and restoration status"
**Expected:** Add explicit cleanup/restoration, tester artifacts, residual risk, and retest fields.

## Example 3: Improve finding quality
**User:** "This report feels weak — make the findings more actionable"
**Expected:** Add evidence, severity/risk rationale, remediation steps, verification guidance, and business context.

## Example 4: Separate audiences
**User:** "Make this usable for execs and engineers"
**Expected:** Build executive-friendly summary plus technical detail that engineers can reproduce.

## Example 5: QA the report before delivery
**User:** "Check if this pentest report is actually complete"
**Expected:** Run completeness, consistency, risk-logic, and sensitive-data handling checks.

## Example 6: Secure delivery mindset
**User:** "What should we do before sending the final report?"
**Expected:** Cover redaction, encryption, distribution control, versioning, and delivery/debrief expectations.

## Example 7: Retest readiness
**User:** "Make the findings easy to retest later"
**Expected:** Ensure reproduction steps, verification guidance, remediation owners, and retest criteria are explicit.
