# User Input Reports

Store user-filled pentest input templates here.

## Naming

Use one of these formats:
- `user-input-YYYY-MM-DD-HHMM.md`
- `<engagement-name>-user-input-YYYY-MM-DD-HHMM.md`

## Rules

- Do not overwrite earlier user submissions unless explicitly instructed.
- Treat each saved file as a source context artifact.
- After saving a filled template, parse it and map entries into the relevant engagement docs and registers.
- Preserve unclassified or low-confidence entries in a traceable way. Do not drop them.
