## Key Intelligence

The `unlock_vault.py` script revealed the attack vector:
```python
def get_pi_serial():
    with open("/proc/cpuinfo", "r") as f:
        for line in f:
            if line.startswith("Serial"):
                return line.split(":")[1].strip()

serial = get_pi_serial()
cmd = f"echo -n '{serial}' | sudo cryptsetup open {vault} nctv_data --key-file -"
```

The `hardware_lock.py` contained the actual authorization values:
```python
AUTHORIZED_PI_SERIAL = "ffb6d42807368154"
AUTHORIZED_SD_CID = "1b534d45423151543089c65df4015a00"
```

## Conclusion

The vault uses hardware serial-based key derivation. The authorized serial (`ffb6d42807368154`) differs from the current device's serial (`dadf6f9ef35e55ab`), indicating the vault was created on a different Pi or the device was swapped.

Next phase: Attempt decryption using discovered authorization values.
