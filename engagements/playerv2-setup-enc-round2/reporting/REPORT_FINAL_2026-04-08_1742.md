## 3. Findings

### V-001 — Encrypted setup container blocks pre-deployment security review

- **Severity:** High
- **CVSS Score:** N/A
- **Affected Target:** engagements/playerv2-artifacts/inbound/setup.enc
- **Description:** The setup artifact is an OpenSSL password-encrypted blob, which prevents direct inspection of installer logic, package type, embedded scripts, or trustworthiness before decryption.
- **Evidence:** ```
Observed file type `openssl enc'd data with salted password`; `Salted__` header present; no plaintext archive signature visible before decryption.
```
- **Impact:** Operators are forced to make trust decisions before they can inspect what will actually run, increasing supply-chain and deployment risk.
- **Remediation:** Require a decrypt-then-review workflow in an isolated analysis directory before installation or execution on Raspberry Pi targets.
- **Hardening:** Treat encrypted setup blobs as opaque containers, not trusted installers. Contents must be reviewed before deployment.
- **References:** setup.enc header: Salted__, SHA-256: f9116a632d11c3b5331e5a9983caff074cc89ec7c8c63d5ac770794ec2d41b7b

### V-002 — Encrypted artifact lacks companion integrity or provenance metadata

- **Severity:** Medium
- **CVSS Score:** N/A
- **Affected Target:** engagements/playerv2-artifacts/inbound/setup.enc
- **Description:** The provided encrypted artifact was not accompanied by a detached signature, signed manifest, or independent checksum package in the current evidence set.
- **Evidence:** ```
Enumeration of `engagements/playerv2-artifacts/` found the encrypted file and quick-scan notes, but no detached trust metadata or release manifest alongside the blob.
```
- **Impact:** A tampered or swapped encrypted installer could be distributed without easy verification of provenance or expected contents.
- **Remediation:** Ship encrypted setup blobs with detached signatures and independently verifiable release metadata.
- **Hardening:** Separate confidentiality from integrity. Encryption should not be the only protection around installer trust.
- **References:** artifact directory listing, enum phase observations

### V-003 — Related Player ecosystem demonstrates hardware-derived key reuse risk

- **Severity:** Medium
- **CVSS Score:** N/A
- **Affected Target:** PlayerV2 setup/deployment trust model
- **Description:** Adjacent Player research in the same environment shows secure storage previously derived decryption directly from Raspberry Pi serial values embedded in scripts. This creates a credible key-reuse and trust-model risk for other protected setup artifacts unless independently disproven.
- **Evidence:** ```
Workspace research under `research/ntv-hardware-lock/` and prior `player-vault` reporting documents serial-derived decryption via `hardware_lock.py` and `unlock_vault.py`, including validated serial `ffb6d42807368154` for the earlier vault flow.
```
- **Impact:** If similar derivation or reuse is applied elsewhere in the setup pipeline, attackers with device-level knowledge may derive or predict protected deployment secrets more easily than intended.
- **Remediation:** Audit `setup.enc` generation workflow to confirm whether any hardware-derived or reused passphrase logic exists. Rotate to unique per-artifact secrets or managed key material if reuse is present.
- **Hardening:** Move encryption key management out of recoverable scripts and device-derived values; use explicit secret custody and release verification instead.
- **References:** research/ntv-hardware-lock/source/.../unlock_vault.py, engagements/player-vault/reporting/REPORT_FINAL_2026-03-18_1800.md

## 4. Security Enhancement Recommendations

### Artifact Review Workflow

Decrypt encrypted installer artifacts only inside a dedicated analysis directory and review contents before any install-time execution.

### Release Provenance

Add detached signatures, signed manifests, and externally verifiable checksums to every protected setup blob.

### Key Management Audit

Review whether `setup.enc` uses device-derived, reused, or script-recoverable decryption material, and eliminate that pattern if found.
