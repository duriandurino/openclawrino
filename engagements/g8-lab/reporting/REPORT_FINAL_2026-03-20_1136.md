## 3. Findings

### G8-001 — SMB Signing Not Required

- **Severity:** Medium
- **CVSS Score:** 5.3
- **Affected Target:** 192.168.0.104:445
- **Description:** SMB2 security mode indicates signing is enabled but not required. In environments where NTLM relay conditions exist, this weakens protection against certain relay and man-in-the-middle scenarios.
- **Evidence:** ```
nmap smb2-security-mode reported: Message signing enabled but not required
```
- **Impact:** If a relay path exists elsewhere in the network, attackers may be able to leverage SMB authentication flows more effectively than on hosts requiring signing.
- **Remediation:** Require SMB signing on the host and through domain or local policy, then verify with smb2-security-mode or equivalent tooling.
- **Hardening:** Reduce unnecessary SMB exposure to trusted admin segments only, disable legacy protocols, and monitor for NTLM relay-style behavior.
- **References:** Microsoft SMB signing guidance

### G8-002 — Multiple Remote Administration Services Exposed

- **Severity:** Medium
- **CVSS Score:** 5.8
- **Affected Target:** 192.168.0.104:135,139,445,3389,5985,7070
- **Description:** The workstation exposes several remote administration and management services including RPC, NetBIOS/SMB, RDP, WinRM, and a likely AnyDesk service over TLS.
- **Evidence:** ```
Nmap detected open services on 135/tcp, 139/tcp, 445/tcp, 3389/tcp, 5985/tcp, and 7070/tcp with Windows/RDP/HTTPAPI/AnyDesk indicators.
```
- **Impact:** This increases the attack surface for credential attacks, lateral movement, remote management abuse, and reconnaissance against an internal workstation.
- **Remediation:** Limit exposed remote management services to those operationally required. Restrict access via host firewall and network segmentation to authorized admin systems only.
- **Hardening:** Require VPN or jump-host access for administration, review AnyDesk necessity, and enforce strong authentication and monitoring on RDP/WinRM.
- **References:** Microsoft RDP hardening guidance, Microsoft WinRM security guidance

### G8-003 — RDP Reveals Domain and Host Metadata

- **Severity:** Low
- **CVSS Score:** 3.7
- **Affected Target:** 192.168.0.104:3389
- **Description:** RDP NTLM information exposed the NetBIOS name, DNS hostname, domain name, and Windows build metadata during enumeration.
- **Evidence:** ```
rdp-ntlm-info returned NetBIOS_Computer_Name=G8, DNS_Computer_Name=G8.nctv360.com, DNS_Domain_Name=nctv360.com, Product_Version=10.0.26100.
```
- **Impact:** This aids attacker reconnaissance and supports tailored credential attacks or environment mapping.
- **Remediation:** Restrict RDP exposure to trusted administration sources and disable it if not needed.
- **Hardening:** Place RDP behind VPN or bastion access, enforce NLA and MFA where possible, and monitor failed logons and RDP connection attempts.
- **References:** CIS Microsoft Windows hardening recommendations

### G8-004 — No Confirmed Legacy SMB Vulnerabilities in Safe NSE Pass

- **Severity:** Info
- **Affected Target:** 192.168.0.104:445
- **Description:** A safe NSE vulnerability pass did not confirm MS10-054 or identify a clear exploitable legacy SMB issue. Some vuln scripts failed negotiation rather than proving exposure.
- **Evidence:** ```
smb-vuln-ms10-054: false; smb-vuln-ms10-061 and samba-vuln-cve-2012-1182 did not negotiate successfully.
```
- **Impact:** This reduces confidence in obvious legacy SMB exposure but does not prove the host is hardened against other auth or configuration issues.
- **Remediation:** Keep SMB fully patched and confirm hardening with authenticated Windows security review if needed.
- **Hardening:** Continue patch management, restrict SMB to trusted peers, and validate with authenticated assessment during a deeper review.
- **References:** Nmap NSE smb-vuln script outputs

## 4. Security Enhancement Recommendations

### Network Segmentation

Restrict workstation administration ports (RDP, WinRM, SMB, AnyDesk) to a dedicated admin subnet or jump host instead of leaving them broadly reachable on the LAN.

### Windows Hardening

Review SMB signing policy, enforce least-privilege remote administration, and disable remote services that are not actively required.

### Monitoring

Alert on repeated RDP, SMB, or WinRM authentication attempts and watch for abnormal remote management access patterns from non-admin systems.
