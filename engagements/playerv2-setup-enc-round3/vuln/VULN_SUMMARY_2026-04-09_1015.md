## Vulnerability Findings

### V-SETUP-004: Encrypted installer remains operationally unverifiable before trust decision
**Severity:** High

Because the artifact is encrypted and still not meaningfully decrypted, reviewers cannot confirm:
- what code or packages it contains
- what services it installs
- whether it creates persistence
- whether it includes unsafe defaults or secrets

This preserves the earlier supply-chain review blind spot.

### V-SETUP-005: Weak verification workflow can create false-confidence decryption events
**Severity:** Medium

The round-3 testing showed a realistic analyst/operator failure mode: a wrong key can return a padding-valid output and appear successful at first glance. Without validating file type, structure, magic bytes, and readability, an operator could falsely conclude the correct passphrase was found.

Impact:
- wasted analysis time
- incorrect reporting
- unsafe execution of garbage output if validation discipline is poor

### V-SETUP-006: Passphrase model remains opaque and nonrecoverable from available evidence
**Severity:** Medium

The artifact still appears dependent on external passphrase knowledge or an unavailable generation workflow. This creates both a security review bottleneck and an operational recovery dependency. If the responsible workflow/script is unavailable, defenders and maintainers cannot independently audit or recover the installer content.
