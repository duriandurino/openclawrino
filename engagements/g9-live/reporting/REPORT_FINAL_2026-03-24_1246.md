## 5. Findings

### CONFIRMED FINDING-001: RDP Legacy TLS Protocol Support

| Field | Details |
|-------|---------|
| **Severity** | LOW |
| **Status** | ✅ Live-Verified |
| **Category** | Cryptographic Configuration |
| **Affected Service** | Remote Desktop Protocol (RDP) on 3389/tcp |

#### Description
The RDP service on G9 is configured to accept connections using deprecated TLS protocol versions 1.0 and 1.1, in addition to modern TLS 1.2 and 1.3. Legacy TLS versions contain known cryptographic weaknesses including BEAST and POODLE attack vectors.

#### Evidence
```
Nmap ssl-enum-ciphers output confirmed:
- TLSv1.0: Enabled with cipher suites
- TLSv1.1: Enabled with cipher suites  
- TLSv1.2: Enabled (secure)
- TLSv1.3: Enabled (secure)
```

#### Impact
- **Confidentiality:** Low — Modern Windows implementations have mitigations for known attacks
- **Integrity:** Low — Message integrity not directly compromised
- **Availability:** None

#### Remediation Steps
1. Disable TLS 1.0 via registry:
   ```
   [HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\SecurityProviders\SCHANNEL\Protocols\TLS 1.0\Server]
   "Enabled"=dword:00000000
   ```
2. Disable TLS 1.1 via registry:
   ```
   [HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\SecurityProviders\SCHANNEL\Protocols\TLS 1.1\Server]
   "Enabled"=dword:00000000
   ```
3. Alternatively, apply via Group Policy: Computer Configuration → Administrative Templates → Network → SSL Configuration Settings
4. Reboot required for changes to take effect

#### Concrete Hardening Recommendations
- Set minimum TLS version to 1.2 for all Windows services
- Enable TLS 1.3 where supported
- Monitor TLS handshake logs for legacy protocol connection attempts
- Consider disabling RDP if not required for business operations
