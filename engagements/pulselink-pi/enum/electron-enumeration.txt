── SECURITY CONSIDERATIONS ───────────────────────────────────────

Electron apps are a rich attack surface because they combine:

1. NODE.JS INTEGRATION: Electron apps typically have Node.js
   integration enabled, meaning renderer processes can make
   system calls. If the app loads untrusted content, this is
   a command injection vector.

2. CHROMIUM ENGINE: inherits all Chrome CVEs. The version of
   Chromium bundled with Electron determines the attack surface
   for web-based exploitation.

3. NODE INTEGRATION IN RENDERER: If nodeIntegration is true
   (default in older Electron), any XSS becomes RCE.

4. CONTEXT ISOLATION: If disabled, renderer can access all
   Node.js APIs. If enabled, limited to exposed preload APIs.

5. PRELOAD SCRIPTS: /opt/electron-player/resources/scripts/
   likely contains preload scripts that run with elevated
   privileges. These are HIGH VALUE targets for finding
   exploitable patterns.

6. ELECTRON UPDATES: Electron apps that auto-update download
   and execute code. If update mechanism is insecure, it's
   a supply chain attack vector.

7. WIDEVINE DRM: The Widevine CDM runs in a sandboxed
   environment but interacts with the app. Known CVEs exist
   for various Widevine versions.
